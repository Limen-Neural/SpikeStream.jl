# Plan: License Migration — GPL-3.0-or-later → Dual MIT/Apache-2.0

**Issue:** [#13](https://github.com/Limen-Neural/SpikeStream.jl/issues/13)
**Priority:** Urgent | **Effort:** Low
**Suggested branch:** `chore/dual-license-mit-apache`

## Context

SpikeStream.jl is the core Julia spike-stream feature extraction package (synchrony, bursts, ISI stats, etc.). The Rust sibling `kinetic-signals` already migrated to dual MIT/Apache-2.0 (issue kinetic-signals#7), and the master tracking issue neuromod#19 is closed. This issue brings SpikeStream.jl into alignment.

The current license is GPL-3.0-or-later in `Project.toml` and the `LICENSE` file. No SPDX headers exist in source files.

---

## Changes

### 1. Replace LICENSE file with dual license files

- **Delete** `LICENSE` (current GPL-3.0 full text)
- **Create** `LICENSE-MIT` — standard MIT license text with copyright `Raul Montoya Cardenas` and current year (2026)
- **Create** `LICENSE-APACHE` — standard Apache License 2.0 full text (SPDX: `Apache-2.0`)

Reference texts: <https://opensource.org/licenses/MIT> and <https://opensource.org/licenses/Apache-2.0>

### 2. Update `Project.toml`

- Change `license = "GPL-3.0-or-later"` → `license = "MIT OR Apache-2.0"`
- Add `license-files = ["LICENSE-MIT", "LICENSE-APACHE"]` (Julia 1.9+ convention)

**File:** `Project.toml`

### 3. Update `README.md`

- Replace the `## License` section with dual license info
